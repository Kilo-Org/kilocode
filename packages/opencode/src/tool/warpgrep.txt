Search the codebase with a natural-language query and return the most relevant code spans.

## When to use

- Find an implementation when you do not know exact filenames or symbol names
- Understand how a feature works in a large unfamiliar repository
- Start broad codebase exploration before narrowing with `Read` or `Grep`
- Search for related examples of a concept, workflow, or behavior
- Run several differently phrased exploratory searches in parallel

## When NOT to use

- Search for exact strings or regex patterns — use `Grep`
- Find files by filename or extension — use `Glob`
- Read the contents of a specific known file — use `Read`
- Search by semantic similarity over indexed snippets — use `SemanticSearch`

## Examples

- "Where is OAuth callback handling implemented?" → find likely feature entry points
- "How does tool approval flow work?" → retrieve relevant code spans across modules
- "Show examples of session resume logic" → search for concept-level matches
- "Find caching code for provider models" → use natural language instead of regex terms

## Constraints

- Write the query in natural language, not as regex or a keyword dump.
- Treat results as read-only context and follow up with `Read` on the returned files when needed.
