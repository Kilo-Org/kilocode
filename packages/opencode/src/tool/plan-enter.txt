Offer a switch to planning mode before implementation work starts.

## When to use

- User explicitly asks for a plan before coding
- Task spans multiple files or subsystems and needs decomposition
- Work requires research, design, or architecture decisions before edits
- You want to separate planning from execution in a dedicated flow

## When NOT to use

- User wants immediate implementation of a simple change — use direct tool calls instead
- Work is already fully planned and ready to execute — use `PlanExit` after finishing the plan
- You only need a quick clarification from the user — use `Question`

## Examples

- "Make me a plan for refactoring auth" → offer planning mode first
- "Before coding, design how this feature should work" → switch to planning flow
- "Map out changes across CLI and extension" → use planning before editing

## Constraints

- Call this before implementation when the user explicitly asks for a plan.
- Use this only to start planning, not to record or finalize a plan.
