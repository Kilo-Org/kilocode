Search file contents with a regular expression and return matching files, line numbers, and line text.

## When to use

- Find known identifiers, strings, or regex patterns across files
- Locate exact API names, flags, or error messages in code or docs
- Filter content search to specific file globs with `include`
- Confirm whether a known pattern still exists after edits
- Narrow large codebases before using `Read`

## When NOT to use

- Find files by filename only — use `Glob`
- Read full contents of a known file — use `Read`
- Explore code by meaning when exact names are unknown — use `SemanticSearch`
- Count every match in detail or use advanced ripgrep flags — use `Bash`
- Delegate long multi-step investigation across many searches — use `Task`

## Examples

- "Find `function\\s+loadConfig`" → regex search for a known function definition
- "Search `TODO|FIXME` in TypeScript files" → use alternation plus `include: *.ts`
- "Look for `log.*Error` under `packages/opencode`" → set `path` to a subdirectory
- "Find imports from `@/kilocode`" → search exact import text across source files

## Constraints

- Provide a regex pattern, not a filename glob.
- Use `include` only to filter which files are searched.
- Use `Bash` with `rg` when you need total match counts instead of file-level hits.
