Replace exact text in an existing file and rewrite the file in place.

## When to use

- Change a focused block in an existing file when you know the current text
- Rename repeated strings across one file with `replaceAll`
- Make surgical edits without replacing the whole file
- Update indentation-sensitive code after reading exact file contents
- Adjust an existing file while preserving the rest of it unchanged

## When NOT to use

- Replace an entire file with new contents — use `Write`
- Make coordinated edits across many files or create/delete files — use `ApplyPatch`
- Read the current file contents first — use `Read`
- Search for candidate files or strings — use `Glob` or `Grep`

## Examples

- "Rename `getCwd` to `getCurrentWorkingDirectory` in one file" → use `replaceAll: true`
- "Change one branch in this function" → pass the exact surrounding block as `oldString`
- "Insert content into an existing file" → replace a nearby anchor string with anchor plus new text
- "Fix indentation-sensitive code from `Read` output" → copy text after the `line: ` prefix only

## Constraints

- Read the file before editing it.
- Copy `oldString` exactly from file contents, excluding the `line: ` prefix added by `Read`.
- Keep `oldString` and `newString` different.
- Provide more surrounding context when `oldString` is not unique, or use `replaceAll`.
- Prefer `Edit` over `Write` for existing files.
