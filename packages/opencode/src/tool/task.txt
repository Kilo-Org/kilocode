Launch a subagent to handle a focused research or implementation task in a separate session.

## When to use

- Delegate complex multistep work that can proceed autonomously
- Run custom slash commands by sending the slash invocation as the prompt
- Split independent investigations or implementations across multiple agents
- Offload broad repository exploration that would take many `Read` and `Grep` calls
- Resume prior delegated work with `task_id`
- Use specialist agents such as code review, library research, or investigation agents

## When NOT to use

- Read a known file or small set of files — use `Read`
- Find files by name or exact symbol definitions quickly — use `Glob` or `Grep`
- Ask the user for missing requirements — use `Question`
- Track your own session task list — use `TodoWrite`

## Examples

- "Research this package and report best practices" → launch a research-oriented subagent with verification guidance
- "Run `/check-file path/to/file.py`" → send the slash command as the full prompt
- "Investigate two unrelated failures" → dispatch separate subagents in parallel
- "Continue previous subagent work" → pass the saved `task_id` to resume that session
- "Review committed branch changes" → call a review-focused subagent and summarize result to user

## Constraints

- Set `subagent_type` to one available agent name.
- Write a self-contained prompt that states scope, expected output, and whether the agent should research or edit code.
- Provide verification commands when the subagent should change code.
- Use separate tool calls in the same message for independent subagents that can run in parallel.
- Pass `task_id` only when resuming an existing subagent session.
