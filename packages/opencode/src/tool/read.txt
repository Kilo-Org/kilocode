Read files or directories from the local filesystem and return numbered text, directory entries, or supported attachments.

## When to use

- Inspect source files, configs, or docs before editing them
- Read a directory listing to confirm paths and nearby files
- Continue through a large file with `offset` and `limit`
- Open images or PDFs as attachments when file inspection is needed
- Read several known files in parallel when they can be understood independently

## When NOT to use

- Find files by name pattern when the path is unknown — use `Glob`
- Search for exact text or regex matches across files — use `Grep`
- Explore code by semantic meaning instead of path — use `SemanticSearch`
- Modify file contents — use `Edit`, `Write`, or `ApplyPatch`

## Examples

- "Read `src/index.ts`" → return numbered lines from start of file
- "Continue this file from line 400" → set `offset: 400`
- "Show contents of `packages/opencode/src/tool/`" → return directory entries with `/` on subdirectories
- "Open this PDF" → return the file as an attachment when format is supported

## Constraints

- Use `offset` as a 1-indexed starting line.
- Read larger windows instead of many tiny repeated slices when more context is needed.
- Expect lines longer than 2000 characters to be truncated.
