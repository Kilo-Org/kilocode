Search the web with Exa for current information and summarized source content.

## When to use

- Look up information beyond model knowledge cutoff
- Find recent news, releases, pricing, or documentation updates
- Compare several web sources in one search result set
- Use live crawling when cached results may be stale
- Gather web context before following up on a specific page with `WebFetch`

## When NOT to use

- Fetch one known URL directly — use `WebFetch`
- Search local repository files — use `Grep`, `Read`, or `SemanticSearch`
- Recall prior local conversations — use `Recall`

## Examples

- "Latest AI news {{year}}" → include current year for recent topics
- "Bun workspace filters documentation" → search current docs across sources
- "Price of service X {{year}}" → use live web data beyond cutoff
- "Deep search for PostgreSQL logical replication changes" → set `type: deep`

## Constraints

- Include the current year `{{year}}` in queries about recent information or current events.
- Use `livecrawl: preferred` only when fresh crawling should take priority over cached results.
- Use `type: fast`, `auto`, or `deep` to match the needed depth.
