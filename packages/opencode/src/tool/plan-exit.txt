Signal that planning is complete and the saved plan is ready for implementation.

## When to use

- Finish a planning session after writing the plan file
- End planning after resolving open questions and trade-offs
- Hand control back once the plan is ready to execute

## When NOT to use

- Start a planning workflow — use `PlanEnter`
- Ask follow-up questions before the plan is settled — use `Question`
- Continue revising an unfinished plan — keep planning instead

## Examples

- "I finished the plan file" → signal readiness with `PlanExit`
- "Planning is done and implementation can begin" → end planning turn
- "All open questions are resolved" → return the plan path and finish planning

## Constraints

- Call this only after the plan file is finalized.
- Do not call this while required implementation questions remain open.
