Apply structured multi-file patches that add, update, move, or delete files.

## When to use

- Make coordinated edits across multiple files in one tool call
- Rename a file while changing its contents with `*** Move to:`
- Create and delete files as part of one atomic patch
- Apply line-level diffs when exact string replacement is awkward
- Preserve reviewable patch structure for large edits

## When NOT to use

- Replace an exact known string in one file — use `Edit`
- Replace an entire file with new contents — use `Write`
- Inspect file contents before editing — use `Read`
- Search for files or symbols to change — use `Glob` or `Grep`

## Examples

- "Add `hello.txt` with one line" → use `*** Add File:` and prefix each content line with `+`
- "Rename `src/app.py` to `src/main.py` while editing it" → use `*** Update File:` with `*** Move to:`
- "Delete `obsolete.txt`" → use `*** Delete File:` with no body
- "Patch several files at once" → include multiple file sections inside one patch envelope

## Constraints

- Wrap every patch in `*** Begin Patch` and `*** End Patch`.
- Start each file section with exactly one header: `*** Add File:`, `*** Update File:`, or `*** Delete File:`.
- Prefix every added line in `*** Add File:` sections with `+`.
- Include at least one file hunk in the patch.
- Use `*** Move to:` only inside an `*** Update File:` section.
