Query configured Language Server Protocol servers for symbol, reference, hover, and call-hierarchy data.

## When to use

- Jump from a symbol use to its definition with `goToDefinition`
- List references or implementations of a symbol
- Inspect hover documentation or type information at a position
- Enumerate symbols in one file with `documentSymbol`
- Search project-wide symbols with `workspaceSymbol`
- Trace callers or callees with call hierarchy operations

## When NOT to use

- Search raw text or regex patterns — use `Grep`
- Read source code around a known path — use `Read`
- Find files by name or extension — use `Glob`
- Explore code when no LSP server is available for the language — use `SemanticSearch` or `Grep`

## Examples

- "Find definition of symbol at `src/app.ts:42:7`" → use `goToDefinition`
- "Show hover docs for this function call" → use `hover`
- "List all symbols in `server.ts`" → use `documentSymbol`
- "Search workspace symbols matching `Config`" → use `workspaceSymbol` with `query`
- "See who calls this method" → use `prepareCallHierarchy` followed by `incomingCalls`

## Constraints

- Provide `line` and `character` as 1-based editor positions.
- Provide a `filePath` for every operation, including `workspaceSymbol`.
- Use `query` only for `workspaceSymbol`; omit it for other operations.
- Call this only for file types with a configured LSP server.
