Create and update a structured todo list for the current session.

## When to use

- Track complex work that has three or more meaningful steps
- Organize multi-part requests or several user goals in one session
- Capture new requirements before starting implementation
- Mark progress as you complete tasks and discover follow-up work
- Make long-running work visible to the user while you execute it

## When NOT to use

- Handle a single trivial task directly — use the relevant tool instead
- Answer an informational question with no execution work — respond without `TodoWrite`
- Read or search codebase content — use `Read`, `Grep`, or `Glob`
- Ask the user for missing requirements — use `Question`

## Examples

- "Implement auth, tests, and docs" → create several concrete todos before editing files
- "Rename this symbol across many files" → track search, edits, verification, and cleanup
- "Run tests after feature work" → add verification as its own todo item
- "Scope expanded mid-task" → update the list immediately with new work

## Constraints

- Keep exactly one todo `in_progress` at a time.
- Use only `pending`, `in_progress`, `completed`, or `cancelled` for status values.
- Mark tasks complete immediately instead of batching status updates later.
- Break complex work into specific actionable items rather than broad themes.
