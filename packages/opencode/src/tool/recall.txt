Search and read past local Kilo conversations for this project and its git worktrees.

## When to use

- Recall how a feature or fix was discussed in an earlier session
- Find prior conversations by session title before reopening them
- Recover context from another worktree in the same repository family
- Review previous assistant and user messages for implementation history
- Check whether related work already happened in another local session

## When NOT to use

- Search current source files for code or text — use `Grep` or `SemanticSearch`
- Read current repository files or docs — use `Read`
- Look up information outside this project family — use `WebSearch` or `WebFetch`

## Examples

- "Find sessions about auth refactor" → use `mode: search` with a title keyword query
- "Open session `abc123`" → use `mode: read` with that `sessionID`
- "See what I did in another worktree" → search first, then read the chosen session
- "Check prior discussion before reimplementing this" → retrieve the earlier transcript

## Constraints

- Use `mode: search` before `mode: read` when you do not already know the session ID.
- Provide `query` for `search` mode.
- Provide `sessionID` for `read` mode.
- Expect access to be limited to the current project or worktree family.
