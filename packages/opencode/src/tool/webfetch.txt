Fetch a specific URL and return its contents as Markdown, plain text, HTML, or an image attachment.

## When to use

- Retrieve content from a known URL for analysis
- Convert HTML pages into Markdown or plain text for easier reading
- Fetch raw HTML when structure matters
- Open remote images as attachments
- Inspect a single page without running a broader web search

## When NOT to use

- Discover relevant websites or recent sources first — use `WebSearch`
- Read local files from disk — use `Read`
- Search repository code or docs — use `Grep`, `Read`, or `SemanticSearch`

## Examples

- "Fetch `https://example.com/docs` as Markdown" → use default `format`
- "Get plain text from this article" → set `format: text`
- "Inspect page source" → set `format: html`
- "Open this image URL" → fetch and return it as an attachment

## Constraints

- Provide a fully formed `http://` or `https://` URL.
- Use `timeout` in seconds and keep it at or below 120.
- Prefer a more targeted web tool when one exists for the task.
