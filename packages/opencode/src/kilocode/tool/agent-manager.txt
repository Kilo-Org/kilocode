Start visible Agent Manager sessions in the VS Code extension, either in new worktrees or in the current workspace.

## When to use

- User explicitly wants work fanned out into Agent Manager
- Start several visible sessions for independent tasks in the extension UI
- Create isolated Agent Manager worktrees for separate implementations
- Launch local Agent Manager sessions that share the current workspace
- Compare alternative versions of the same work with grouped sessions

## When NOT to use

- Delegate ordinary hidden subagent work — use `Task`
- Track work only in the current session — use `TodoWrite`
- Create git worktrees outside Agent Manager — use regular repository tools instead

## Examples

- "Start three Agent Manager worktrees for these bug fixes" → use `mode: worktree` with three tasks
- "Open two local Agent Manager sessions in this repo" → use `mode: local`
- "Generate two versions of same feature for comparison" → set `versions: true`
- "Start a session with only a branch name seed" → provide `branchName` in the task entry

## Constraints

- Use this only when the user wants visible Agent Manager sessions in the VS Code extension.
- Set `mode` to either `worktree` or `local`.
- Provide between 1 and 20 tasks.
- Give each task at least one of `prompt`, `name`, or `branchName`.
- Set `versions: true` only when tasks are alternate versions of the same work.
