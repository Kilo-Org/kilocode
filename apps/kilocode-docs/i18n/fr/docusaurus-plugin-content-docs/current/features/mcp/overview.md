# Protocole de Contexte Modèle (MCP)

Le Protocole de Contexte Modèle (MCP) est une norme pour étendre les capacités de Kilo Code en se connectant à des outils et services externes. Les serveurs MCP fournissent des outils et ressources additionnels qui aident Kilo Code à accomplir des tâches au-delà de ses capacités intégrées, comme l'accès aux bases de données, APIs personnalisées, et fonctionnalités spécialisées.

## Documentation MCP

Cette documentation est organisée en plusieurs sections :

- [**Utiliser MCP dans Kilo Code**](/features/mcp/using-mcp-in-kilo-code) - Guide complet pour configurer, activer et gérer les serveurs MCP avec Kilo Code. Inclut les paramètres serveur, l'approbation d'outils, et le dépannage.

- [**Qu'est-ce que MCP ?**](/features/mcp/what-is-mcp) - Explication claire du Protocole de Contexte Modèle, son architecture client-serveur, et comment il permet aux systèmes IA d'interagir avec des outils externes.

- [**Transports STDIO & SSE**](/features/mcp/server-transports) - Comparaison détaillée des mécanismes de transport locaux (STDIO) et distants (SSE) avec les considérations de déploiement pour chaque approche.

- [**MCP vs API**](/features/mcp/mcp-vs-api) - Analyse de la distinction fondamentale entre MCP et les APIs REST, expliquant comment ils opèrent à différentes couches d'abstraction pour les systèmes IA.
