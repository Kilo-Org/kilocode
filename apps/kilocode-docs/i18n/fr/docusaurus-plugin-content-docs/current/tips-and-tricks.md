# Astuces & Conseils

Une collection de conseils rapides pour vous aider à tirer le meilleur parti de Kilo Code.

- Glissez Kilo Code vers la [Barre Latérale Secondaire](https://code.visualstudio.com/api/ux-guidelines/sidebars#secondary-sidebar) pour pouvoir voir l'Explorateur, la Recherche, le Contrôle de Source, etc.
- Une fois que vous avez Kilo Code dans une barre latérale séparée de l'explorateur de fichiers, vous pouvez glisser des fichiers de l'explorateur dans la fenêtre de chat (et même plusieurs à la fois). Assurez-vous simplement de maintenir la touche shift enfoncée après avoir commencé à glisser les fichiers.
- Si vous n'utilisez pas [MCP](/features/mcp/overview), désactivez-le dans l'onglet <Codicon name="notebook" /> Prompts pour réduire considérablement la taille du prompt système.
- Pour garder vos [modes personnalisés](/features/custom-modes) sur la bonne voie, limitez les types de fichiers qu'ils sont autorisés à modifier.
- Si vous rencontrez l'erreur redoutée `input length and max tokens exceed context limit`, vous pouvez récupérer en supprimant un message, en revenant à un point de contrôle précédent, ou en basculant vers un modèle avec une fenêtre de contexte longue comme Gemini pour un message.
- En général, soyez réfléchi concernant votre paramètre `Max Tokens` pour les modèles de réflexion. Chaque token que vous allouez à cela réduit l'espace disponible pour stocker l'historique de la conversation. Envisagez d'utiliser des paramètres `Max Tokens` / `Max Thinking Tokens` élevés uniquement avec des modes comme Architect et Debug, et de garder le mode Code à 16k tokens maximum ou moins.
- S'il y a une offre d'emploi réelle pour quelque chose que vous voulez qu'un mode personnalisé fasse, essayez de demander au mode Code de `Créer un mode personnalisé basé sur l'offre d'emploi à @[url]`
- Si vous voulez vraiment accélérer, consultez plusieurs copies de votre dépôt et exécutez Kilo Code sur toutes en parallèle (en utilisant git pour résoudre les conflits, comme avec les développeurs humains).
- Lors de l'utilisation du mode Debug, demandez à Kilo de "démarrer une nouvelle tâche en mode Debug avec tout le contexte nécessaire pour comprendre X" afin que le processus de débogage utilise sa propre fenêtre de contexte et ne pollue pas la tâche principale
- Ajoutez vos propres conseils en cliquant "Modifier cette page" ci-dessous !
- Pour gérer les fichiers volumineux et réduire l'utilisation de contexte/ressources, ajustez le paramètre `File read auto-truncate threshold`. Ce paramètre contrôle le nombre de lignes lues depuis un fichier en un seul lot. Des valeurs plus faibles peuvent améliorer les performances lors du travail avec des fichiers très volumineux, mais peuvent nécessiter plus d'opérations de lecture. Vous pouvez trouver ce paramètre dans les paramètres de Kilo Code sous 'Paramètres Avancés'.
