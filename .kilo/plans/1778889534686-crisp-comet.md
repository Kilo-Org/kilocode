# Plan: Show Running Status Indicator in Subagent Footer

## Problem

When viewing a subagent (child) session in the TUI, there is no visual indicator showing whether the subagent is currently running or idle. The main session shows an animated spinner + "esc interrupt" hint in the prompt component when `status().type !== "idle"`, but the subagent footer only shows the subagent label, sibling count, usage stats, and navigation buttons.

## Solution

Add a running status indicator to the `SubagentFooter` component that shows an animated spinner when the subagent session is actively working, without the "esc interrupt" hint (since subagents don't have an input prompt to interrupt).

## Implementation Details

### File: `packages/opencode/src/cli/cmd/tui/routes/session/subagent-footer.tsx`

1. **Import `Spinner` component** from `@tui/component/spinner`

2. **Add running status computation** using the same approach as the main session:
   - Create a memo `isRunning` that checks the session status from `sync.data.session_status[route.sessionID]`
   - The status is `"busy"` when the agent is working, `"idle"` when done
   - Also check via messages: if the last message is an assistant message without `time.completed`, the subagent is running

3. **Render the spinner** in the footer's left section (after the subagent label, before usage stats):
   - Show `<Spinner />` when `isRunning()` is true
   - Color the spinner using the subagent's agent color (via `local.agent.color()`)
   - No "esc interrupt" text — just the spinner animation

### Specific Changes

```tsx
// Add imports
import { Spinner } from "@tui/component/spinner"
import { useLocal } from "@tui/context/local"

// Inside SubagentFooter component, add:
const local = useLocal()

const isRunning = createMemo(() => {
  const status = sync.data.session_status?.[route.sessionID]
  if (status?.type === "busy") return true
  // Fallback: check if last assistant message is incomplete
  const msg = messages()
  const last = msg.findLast((m) => m.role === "assistant")
  if (last && !last.time.completed) return true
  return false
})

const agentColor = createMemo(() => {
  const s = session()
  return local.agent.color(s?.agent ?? "")
})
```

In the JSX, insert the spinner between the subagent label/count and the usage stats:

```tsx
<box flexDirection="row" gap={1}>
  <text fg={theme.text}>
    <b>{subagentInfo().label}</b>
  </text>
  <Show when={subagentInfo().total > 0}>
    <text style={{ fg: theme.textMuted }}>
      ({subagentInfo().index} of {subagentInfo().total})
    </text>
  </Show>
  {/* NEW: Running indicator */}
  <Show when={isRunning()}>
    <Spinner color={agentColor()} />
  </Show>
  <Show when={usage()}>
    ...
  </Show>
</box>
```

### Key Design Decisions

- **Spinner only, no "esc interrupt"**: Subagent sessions don't have an input prompt, so there's nothing to interrupt. The spinner alone communicates "working" status.
- **Agent-colored spinner**: Uses the same color-coding as the main session's spinner to visually tie the activity to the specific subagent type.
- **Dual status check**: Checks both `session_status` store (real-time via SSE events) and message completion state as a fallback for accuracy.
- **Placement**: Spinner appears after the label/count but before usage stats, keeping the existing layout clean.
