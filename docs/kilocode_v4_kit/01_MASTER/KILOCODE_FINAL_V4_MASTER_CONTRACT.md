# KiloCode Full Coverage Final v4 Master Contract

Date: 2026-04-17

## Mission
Build KiloCode into a full-coverage, enterprise-grade, dark-first agent cockpit with complete coverage for:
- SSH and remote systems
- VPS and infrastructure operations
- Hermes interoperability
- ZeroClaw execution delegation
- provider routing and fallback
- Shiba memory visibility
- training and GPU orchestration
- governance, approvals, auditability, release, and rollback

## End-state definition
KiloCode must function as all of the following:

1. IDE and workspace cockpit
2. remote systems console
3. governed execution surface
4. provider-routing control surface
5. memory-aware agent workbench
6. training orchestration dashboard
7. release and rollback control center

## Core architecture
- Discord/front-door bot = user entry surface
- Hermes = planner, router, memory coordinator, authority engine
- KiloCode = cockpit and workspace
- ZeroClaw = hardened execution substrate
- Providers = intelligence pool
- Shiba = continuity and recall layer
- Truth docs + ledgers = audit substrate

## Non-negotiables
- no fake completion
- no stale docs after runtime/config change
- no phase marked complete without evidence
- no high-risk execution without approval path
- no routing claims without trace proof
- no memory claims without visible read/write evidence
- no training claims without launch/monitor/resume/compare evidence
- no release claims without rollback evidence

## Global completion standard
The project is only complete when all of the following are true:
- all 72 phases passed
- all critical truth matrix rows complete
- no unresolved critical defects
- all subsystem acceptance criteria pass
- owner + confirmer + challenger workflow followed
- at least 3 end-to-end workflows pass
- final release verdict is PASS or CONDITIONAL PASS with zero critical blockers

## Required outputs
- phase tracker
- feature truth matrix
- defect ledger
- run ledger
- evidence bundle paths
- acceptance reports
- release verdict
- deployment checklist
- rollback checklist

## Hard stop triggers
Stop immediately if any of the following occur:
- docs and runtime disagree
- provider routing is inexplicable
- approval path can be bypassed
- ZeroClaw can execute outside allowed scope
- memory path produces false positive “success”
- training job claims exist without real job lifecycle proof
- release path lacks rollback instructions
