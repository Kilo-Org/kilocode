## 1. 目标与原则

### 1.1 产品目标

面向国企、军工、航天、金融等高安全场景，基于 Kilo Code Fork 交付 **代码不出域** 的企业级 AI 编程工具（合同 §2.1～§2.2）。

### 1.2 合同约束原则

| 原则 | 合同依据 | 对开发计划的影响 |
|---|---|---|
| 固定总价、三批次交付 | §3.2、§4.2 | Phase 1/2/3 范围 **不可擅自缩减**；变更走 §12 需求变更 |
| 验收驱动 | §5.2～§5.3 | 每批次交付前对照 **功能 + 性能 + 安全 + 文档** 四类标准 |
| 知识产权归甲方 | §6.1 | 全部交付物入甲方指定 Git 私有仓 |
| 数据不出域 | §8.1、§16.3 | 开发/测试禁止代码上传乙方或公网云 |
| 开源合规 | §6.2、§16.1 | **T+2 周**提交《开源组件合规报告》；Kilo Apache 2.0 关于页保留 |
| 等保/密评设计 | §8.2、§16.2 | 代码层满足等保 2.0 三级、密评三级；测评费用由甲方另付 |
| 代码质量 | §10.4 | 注释率 ≥20%；单测覆盖率 ≥60%；SonarQube 无阻断 |

### 1.3 技术原则（架构图补充）

| 原则 | 说明 |
|---|---|
| 防腐桥接 | L2 五模块吸收 Kilo 升级；L3 经虚线控制面影响 L2 |
| 网关统一入口 | 生产路径：L5 → L4 鉴权 → L3/L2 → L1（Phase 1 验收需覆盖 L1～L5 **基础联调**） |
| Engine / CLI 分工 | Engine = `kilo serve`；CLI = Agent Loop + 代码索引 |
| Fork 独立维护 | 企业逻辑优先 `enterprise/`、`deploy/enterprise/`；L3 独立仓 |

## 5. 验收标准摘要（合同 §5.2）

按附件一 v2.4 + **附件三《验收标准详细说明》**（待甲方确认）逐条用例验证。

### 5.2 性能验收（M4 前必须达标）

| 指标 | 标准 |
|---|---|
| 代码补全流式 P95 | **&lt; 3 秒**（14B，**100 并发**） |
| 对话/Agent P95 | **&lt; 5 秒**（80B，**50 并发**） |
| 可用性 | **≥ 99.9%** |
| 错误率 | **&lt; 0.1%** |

### 5.3 安全验收

| 项 | 标准 |
|---|---|
| 静态/镜像扫描 | Semgrep、Trivy：**高危 = 0**；中危 ≤5 且附修复计划 |
| 依赖许可证 | 无 GPL/LGPL 传染性风险（或已隔离） |
| 敏感信息 | 代码提交前密钥扫描（§8.1.3） |
