## 1. 通用验收规则

| 类型 | 通过条件 |
|---|---|
| 功能 | 本附件用例 **100% 执行**；关键用例 **100% 通过**；一般用例通过率 ≥95% |
| 性能 | 见 §5；在甲方或双方确认的 GPU/并发环境下实测 |
| 安全 | Trivy/Semgrep 扫描；依赖许可证清单无未披露 GPL 传染 |
| 文档 | 交付物清单与合同 §1.2 一致；步骤可复现 |

**有条件通过：** 非 blocker 项限期整改（5 工作日）；**不通过：** blocker 未关闭或连续两次未达标。

## 2. Phase 1 验收（M2，T+6 周）

**合同交付概要：** L1 容器化；L2 桥接骨架；L4 APISIX；L5 品牌；License 在线 + **离线 RSA 原型**；L1～L5 基础联调。

### 2.1 Layer 1 — 容器化

| ID | 用例 | 步骤 | 预期 | 证据 |
|---|---|---|---|---|
| P1-L1-01 | Engine 健康检查 | `curl -u kilo:$PWD http://host:4096/global/health` | HTTP 200 | 截图/日志 |
| P1-L1-02 | Compose 启动 | `docker compose up -d kilo-engine qdrant` | 两容器 Running | `docker ps` |
| P1-L1-03 | Qdrant 可达 | `curl http://host:6333/healthz` | HTTP 200 | 日志 |
| P1-L1-04 | 可选 vLLM profile | `compose --profile vllm up -d`（有 GPU 时） | vLLM 容器 Running | 部署记录 |

### 2.2 Layer 2 — 桥接骨架

| ID | 用例 | 步骤 | 预期 | 证据 |
|---|---|---|---|---|
| P1-L2-01 | Bridge 健康 | `curl http://host:8080/health` | 返回 `ok` | 日志 |
| P1-L2-02 | Bridge 代理 Engine | `curl -u kilo:$PWD http://host:8080/global/health` | HTTP 200 | 日志 |
| P1-L2-03 | 版本适配框架 | 请求带 `X-Enterprise-Api-Version: v0` | 响应头含 `X-Bridge-Normalized-Version` | bridge 日志 |
| P1-L2-04 | OpenAPI 草案 | 打开 `deploy/enterprise/openapi.yaml` | 含 License、健康检查路径 | 文档 |

### 2.3 Layer 4 — APISIX

| ID | 用例 | 步骤 | 预期 | 证据 |
|---|---|---|---|---|
| P1-L4-01 | 网关链路透传 | `curl -u kilo:$PWD http://host:9080/kilo/global/health` | HTTP 200（经 bridge → engine） | smoke 脚本 |
| P1-L4-02 | 限流 | 1 分钟内同一 IP 请求 >100 次 | 出现 HTTP 429 | 压测记录 |

### 2.5 License 原型

| ID | 用例 | 步骤 | 预期 | 证据 |
|---|---|---|---|---|
| P1-LC-01 | 在线校验 | POST `/api/v1/license/verify`（mock 或网关路由） | `valid: true` | curl 输出 |
| P1-LC-02 | 离线 RSA | 使用 `samples/offline-license.signed.json` + `license-dev-public.pem` | 插件 reason=`offline_rsa` | 截图/日志 |
| P1-LC-03 | 签名失败 | 篡改 signature 字段 | 阻断，`offline_bad_signature` | 单测/手动 |
| P1-LC-04 | 到期阻断 | expiresAt 设为过去时间 | 阻断连接 | 单测/手动 |
| P1-LC-05 | 未授权阻断 | 无有效 License | 不连接 Engine | 截图 |

## 6. Blocker 定义（Phase 1）

以下任一项未通过则 M2 **不得**判为通过：

1. P1-LC-02 离线 RSA 验签原型  
2. P1-L2-01～02 桥接代理 Engine  
3. P1-L4-01 网关全链路 health  
4. P1-L5-04 或 P1-L5-05 至少一种连接方式可用  
5. P1-DOC-01、P1-DOC-04
