Component structure:
- Phase-3 dual-branch pattern: `useRenderTarget()` to choose DOM vs terminal-stub branch
- DOM branch: `<table>` with 6 `<th>` headers and 1 `<tr>` per role
- Terminal stub: `<text>` summarizing rows + Phase 5 TODO

Required props:
```tsx
import type { ZodIssue } from "zod"
import type { CanonicalPosition } from "@devilcode/cli/devilcode/team/library"
import type { CanonicalTeamRole } from "@devilcode/cli/devilcode/team/config"

export type RosterTableProps = {
  roles: Record<string, CanonicalTeamRole>     // keyed by CanonicalPosition string
  errorsByRole: Record<string, ZodIssue[]>
  onEdit(positionId: CanonicalPosition, field: keyof CanonicalTeamRole, value: unknown): void
  onDelete(positionId: CanonicalPosition): void
  onAdd(): void                                 // opens picker — caller controls
  selectedRole?: CanonicalPosition | null       // optional: visually highlight focused row
  onSelectRole?(positionId: CanonicalPosition): void
}
```

DOM branch must:
1. Render a `<table>` with `<thead>` containing 6 `<th>` cells in this exact order: `Position`, `Provider`, `Model`, `Effort`, `Delegates-to`, `Capabilities`.
2. For each entry in `roles`, render `<tr data-position={positionId} data-has-errors={hasErrors}>` with cells:
   - **Position** — `<td>{role.displayName}</td>` (also show `positionId` as small subtitle)
   - **Provider** — `<input type="text" value={role.provider} onInput=... onChange=...>` calling `onEdit(positionId, "provider", e.currentTarget.value)`
   - **Model** — same pattern, field `"model"`
   - **Effort** — `<select>` with options from `EffortLevel.options` (import from `team/effort.ts` or `team/config.ts`); on change call `onEdit(positionId, "effort", value)`
   - **Delegates-to** — comma-separated string of `role.canDelegate` rendered via `<input>` (Phase 6 polishes to multi-select); on change parse `value.split(",").map((s) => s.trim()).filter(Boolean)` and call `onEdit(positionId, "canDelegate", parsed)`
   - **Capabilities** — render each capability as a small `<span>` chip with role-default color; non-editable in Phase 4 (clicking chip = noop with cursor: not-allowed; Phase 6 polishes to add/remove)
3. After the row cells, append a small `<button data-action="delete" onClick={() => props.onDelete(positionId)}>×</button>` aligned right.
4. Below the `<tbody>`, render `<button data-action="add" onClick={() => props.onAdd()}>+ Add Position</button>`.
5. Apply visual error state to rows where `props.errorsByRole[positionId]?.length > 0`:
   - Row background: `#3a1a1a`
   - Border: `1px solid #a33`
   - Tooltip via `title={errors.map(e => e.message).join("\n")}`
6. If `props.selectedRole === positionId`, add `data-selected="true"` and a left border accent (`border-left: 3px solid #5fa8ff`).
7. Click handler on `<tr>`: `onClick={() => props.onSelectRole?.(positionId)}` (only if callback provided).
