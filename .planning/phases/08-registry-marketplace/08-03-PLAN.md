## Security Threat Model (from spec)
- Signature forgery: Attacker creates manifest with valid-looking signature
- Manifest tampering: MITM modifies envelope/metadata post-signing
- Trust store poisoning: Attacker adds malicious publisher key
- Replay attacks: Old signed manifest installed after revocation
- Path traversal: Malformed manifest paths escape install directory

   # Team Registry & Sharing
   
   Share team configurations with signed manifests for authenticity verification.
   
   ## Quick Start
   
   ### Publishing a Team
   ```bash
   # Export your current team as a signed manifest
   team publish ./my-team-manifest.json --name="My Team" --author="Your Name" --version="1.0.0" --sign=./my-private-key.pem
   ```
   
   ### Installing a Team
   ```bash
   # Install from URL
   team install https://example.com/teams/my-team-manifest.json
   
   # Install from local file
   team install ./downloaded-manifest.json
   ```
   
   ## Generating Keys
   
   Generate an Ed25519 key pair for signing:
   ```bash
   openssl genpkey -algorithm Ed25519 -out private-key.pem
   openssl pkey -in private-key.pem -pubout -out public-key.pem
   ```
   
   ## Trust Management
   
   ### Adding a Trusted Publisher
   ```bash
   team trust ./publisher-public-key.pem publisher-uuid-here
   ```
   
   ### Removing a Trusted Publisher
   ```bash
   team untrust publisher-uuid-here
   ```
   
   ## Manifest Format
   
   [Document the TeamRegistryManifest structure]
   
   ## Security Model
   
   - Ed25519 signatures verify manifest authenticity
   - Explicit key pinning (no trust-on-first-use)
   - Unsigned manifests show warnings but can still be installed
   - Trust store at `~/.local/share/kilo/registry/trusted-publishers.json`
   
   ## Troubleshooting
   
   [Common errors and solutions]
   ```
