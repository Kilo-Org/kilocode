## Task 2: Dashboard Container and Charts

Create the telemetry dashboard components.

**Action:**
1. Create directory `packages/devil-vscode/webview-ui/agent-manager/dashboards/`
2. Create `dashboards/index.tsx` — container with 4 dashboard slots:
   ```typescript
   export function TelemetryDashboards(): JSX.Element {
     const [data, setData] = createSignal<AggregationResponse | null>(null)
     const [loading, setLoading] = createSignal(false)
     
     const refresh = () => {
       setLoading(true)
       postMessage({ type: "teamBuilder.getAggregations" })
     }
     
     // Listen for teamBuilder.aggregations message
     
     return (
       <div class="dashboards-grid">
         <SuccessRateChart data={data()?.successRateByTeam} />
         <StallRateChart data={data()?.stallRateByPosition} />
         <CostChart data={data()?.costByWorkflow} />
         <DurationChart data={data()?.durationByStage} />
         <button onClick={refresh}>Refresh</button>
       </div>
     )
   }
   ```
3. Create `dashboards/SuccessRateChart.tsx`:
   - Bar chart showing completed/started ratio per team
   - Plain SVG rendering (no charting library)
   - Empty state when no data
4. Create `dashboards/StallRateChart.tsx`:
   - Horizontal bars showing max wait time per position
   - Color coding for bottlenecks (red > 60s, yellow > 30s)
5. Create `dashboards/CostChart.tsx`:
   - Simple list or bar showing cost per workflow
   - "Cost data unavailable" message when all zeros
6. Create `dashboards/DurationChart.tsx`:
   - Stacked bar showing avg duration per stage
   - 7 segments for 7 workflow stages
7. Add "Telemetry" tab to `AgentManagerApp.tsx` tab list

## Task 3: SDK Type Exports and CI Compliance

Export team types from SDK and verify CI compliance.

**Action:**
1. Create `packages/sdk/js/src/team.ts` with MANUAL type definitions:
   ```typescript
   // Re-export team types for external consumers
   // NOTE: These are manual type definitions mirroring CLI Zod schemas.
   // No generate.ts exists — types are defined directly here.
   // Keep in sync with packages/opencode/src/devilcode/team/config.ts
   
   // Position library type (mirrors library.ts)
   export interface CanonicalPosition {
     id: string
     displayName: string
     description: string
     defaultCapabilities: string[]
     tier: number
     canDelegate: boolean
   }
   
   // Team config type (mirrors config.ts CanonicalTeamConfig)
   export interface CanonicalTeamConfig {
     name?: string
     roles: CanonicalTeamRole[]
     routing?: CanonicalTeamRouting
     workflowOverride?: DAGOverride
   }
   
   export interface CanonicalTeamRole {
     positionId: string
     displayName?: string
     provider: string
     model: string
     effort: "low" | "medium" | "high" | "max"
     tier?: number
     canDelegate?: boolean
     capabilities?: string[]
   }
   
   export interface CanonicalTeamRouting {
     strategy?: "round-robin" | "load-balanced" | "primary-fallback"
     defaultRole?: string
     parentRole?: string
     reviewEscalationRole?: string
   }
   
   export interface DAGOverride {
     stages?: string[]
     capabilityOverrides?: Record<string, string[]>
   }
   
   export interface TeamHandle {
     id: string
     name: string
     isQuickstart: boolean
   }
   
   // Aggregation response type (matches CLI aggregations.ts)
   export interface AggregationResponse {
     successRateByTeam: Record<string, { completed: number; started: number; rate: number }>
     stallRateByPosition: Record<string, { maxWaitMs: number; avgWaitMs: number }>
     costByWorkflow: Array<{ workflowId: string; totalCost: number }>
     durationByStage: Record<string, { avgMs: number; p95Ms: number; count: number }>
     generatedAt: string
   }
   ```
2. Update SDK barrel export (`packages/sdk/js/src/index.ts`) to include team exports
3. NO generator script — types are manually maintained. Add JSDoc note to keep in sync with CLI.
4. **CYCLE 2 (R1):** Add type validation test at `test/devilcode/sdk/team-types.test.ts`:

   - Assert SDK type shapes match Zod inferred types
   - This catches drift at compile time
   - Example: `expectTypeOf<SDKCanonicalTeamConfig>().toEqualTypeOf<z.infer<typeof CanonicalTeamConfig>>()`
4. Run CI checks:
   - `cd packages/devil-vscode && bun run knip` — verify no dead exports
   - `cd packages/devil-vscode && bun run format:check` — verify formatting
   - `bun turbo typecheck` — verify all types resolve
5. Fix any knip or format issues before completing
