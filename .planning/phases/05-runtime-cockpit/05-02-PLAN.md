### HelpOverlay terminal branch

Group keybinds by section/context and render as text grid:

```tsx
function TerminalBranch(props: HelpOverlayProps) {
  const keybinds = useKeybindRegistry()  // or accept as prop depending on existing shape

  const grouped = createMemo(() => {
    const byGroup: Record<string, typeof keybinds[]> = {}
    for (const k of keybinds()) {
      const group = k.context ?? "global"
      ;(byGroup[group] ??= []).push(k)

      <text><b>Keybindings (ESC to close)</b></text>

/**
 * OnboardingWizard — 3-step first-run onboarding modal.
 *
 * Steps:
 *   1. pick     — quickstart template list + PositionPicker
 *   2. review   — read-only RosterTable + StageCoverageIndicator; Start button gated on useTeamValidation
 *   3. save     — performs save + startBuild (caller handles persistence via onReviewAccept)
 *

import { createSignal, createMemo, Show, For, onMount, type JSX } from "solid-js"
import { useRenderTarget } from "../../context/render-target"
import { useTeamValidation } from "../../hooks/use-team-validation"
import { RosterTable } from "../../components/roster-table"
import { PositionPicker } from "../../components/position-picker"
import { StageCoverageIndicator } from "../stage-coverage-indicator"
import type { CanonicalTeamConfig } from "@devilcode/cli/devilcode/team/config"

export type OnboardingWizardStep = "pick" | "review" | "save" | "done"

export type QuickstartEntry = {
  id: string
  displayName: string
  description?: string
}

export type OnboardingWizardProps = {
  open: boolean
  quickstarts: QuickstartEntry[]
  /** Returns the CanonicalTeamConfig for the picked quickstart. Must be synchronous or Promise. */
  onLoadQuickstart: (id: string) => CanonicalTeamConfig | Promise<CanonicalTeamConfig>
  /** Called when user accepts review. Caller handles save + startBuild + markFirstRunComplete. */
  onReviewAccept: (config: CanonicalTeamConfig) => Promise<void>
  onCancel: () => void
}

export function OnboardingWizard(props: OnboardingWizardProps): JSX.Element {
  const target = useRenderTarget()
  const [step, setStep] = createSignal<OnboardingWizardStep>("pick")
  const [draft, setDraft] = createSignal<CanonicalTeamConfig | undefined>(undefined)
  const [saving, setSaving] = createSignal(false)
  const [error, setError] = createSignal<string | undefined>(undefined)

  const validation = useTeamValidation(() => draft() ?? {})

  async function handlePick(quickstartId: string): Promise<void> {
    try {
      const config = await props.onLoadQuickstart(quickstartId)
      setDraft(config)
      setStep("review")
    } catch (err) {
      setError(err instanceof Error ? err.message : String(err))
    }
  }

  async function handleAccept(): Promise<void> {
    const config = draft()
    if (!config || !validation().isValid) return
    setSaving(true)
    setError(undefined)
    try {
      await props.onReviewAccept(config)
      setStep("done")
    } catch (err) {
      setError(err instanceof Error ? err.message : String(err))
    } finally {
      setSaving(false)
    }
  }

  return (
    <Show when={props.open}>
      <Show
        when={target.kind === "dom"}
        fallback={() => (
          <TerminalBranch
            step={step()}
            quickstarts={props.quickstarts}
            draft={draft()}
            validation={validation()}
            saving={saving()}
            error={error()}
            onPick={handlePick}
            onAccept={handleAccept}
            onCancel={props.onCancel}
          />
        )}
      >
        <DomBranch
          step={step()}
          quickstarts={props.quickstarts}
          draft={draft()}
          validation={validation()}
          saving={saving()}
          error={error()}
          onPick={handlePick}
          onAccept={handleAccept}
          onCancel={props.onCancel}
        />
      </Show>
    </Show>
  )
}

### 3b. Create `packages/devil-ui/CONVENTIONS.md`

Location: repository-standard convention doc lives alongside `package.json` in the package root (NOT inside src/). Per Phase 3 precedent (`TERMINAL-STORYBOOK-DECISION.md` is in src/primitives/), follow the closest pattern: `packages/devil-ui/CONVENTIONS.md`.

Content (minimum 40 lines):

```markdown
# devil-ui Conventions

Effective 2026-04-19 (Phase 5 onwards). Codifies patterns that have repeatedly caused regressions in Phases 3 + 4.

## 1. `<Show fallback>` MUST use the lazy form

```tsx
// ❌ Do NOT: eager JSX construction — the TerminalStub subtree is instantiated on every render
// even when target.kind === "dom" and the fallback is never shown
<Show when={target.kind === "dom"} fallback={<TerminalStub foo={bar} />}>

// ✅ DO: thunk form — SolidJS lazily evaluates only if the when predicate is false
<Show when={target.kind === "dom"} fallback={() => <TerminalStub foo={bar} />}>
```

Rationale: eager eval was safe in Phase 4 (terminal stubs were stateless <text> only), but Phase 5 introduces reactive terminal branches (TabGroup keyboard handlers, DetailPanel with `useDensity()` subscriptions). Eager eval re-constructs those subtrees and attaches listeners on every render regardless of the target branch.

## 2. ARIA boolean attributes MUST be string-typed

WAI-ARIA 1.2 requires `aria-selected`, `aria-pressed`, `aria-invalid`, `aria-expanded`, `aria-checked` to be `"true"` or `"false"` — NOT raw JS booleans. Browsers tolerate booleans but assistive tech and static analyzers do not.

```tsx
// ❌ Do NOT
<button aria-pressed={isActive}>

// ✅ DO
<button aria-pressed={isActive ? "true" : "false"}>
```

## 3. No `@opentui/*` static imports outside primitive terminal branches and adapters

devil-ui consumers (the VS Code webview in Phase 9) never load OpenTUI. Static imports of `@opentui/solid` or `@opentui/core` at the top level of a DOM-branch component pull OpenTUI into the webview bundle.

Allowed locations:
- `src/adapters/terminal.ts` — explicit dynamic-import factory.
- Primitive files' `TerminalBranch` function body — fine because the branch only renders if `target.kind === "terminal"`.
- Never: hook bodies, context providers, or DOM branches.

## 4. PasteModal single-line caveat

OpenTUI has no `<textarea>` element. The PasteModal terminal branch ships a single-line `<input>`; multi-line paste relies on terminal emulator paste (which OpenTUI will forward as one burst). This is a documented accepted limitation through Phase 5; re-evaluate when OpenTUI upstream ships `<textarea>`.

## 5. Test harness

Component + hook tests use the Phase 3 `withRoot` harness (Bun native runner; no `@solidjs/testing-library`). Where JSX mounting is constrained by `@opentui/solid`, fall back to structural smoke tests that assert export shape + inline style/ARIA strings (Phase 4 precedent at `test/devilcode/workflow-tui/index.smoke.test.ts`).

## 6. Storybook branch

Only DOM branches get stories (terminal Storybook INFEASIBLE per `src/primitives/TERMINAL-STORYBOOK-DECISION.md`). Terminal coverage lives in `*-terminal.test.ts` structural tests.
```
