Quickstart authoring rule (resolves stage-coverage gap + tier drift):

Running `fromLegacyTeamConfig` on the 5 presets produces coverage gaps because canonical roles have fewer/different capabilities than the original stringly-typed ones. Additionally, `fromLegacyTeamConfig` sets `tier = POSITION_LIBRARY[positionId].tier`, overriding the legacy role's tier. Hand-authored quickstart JSONs MUST:

1. Close every stage coverage gap by adding minimal canonical roles.
2. Preserve authorial intent on tier (primary vs subagent) — a role that was tier 1 in legacy AND whose mapped canonical position's library-default tier is also 1 (e.g., `senior-dev`, `architect`, `coordinator`) stays tier 1. A role that was tier 1 in legacy but maps to a canonical position whose library tier is 2 (e.g., `coder` → `developer` which is tier 2) must be hand-tuned: either promote the quickstart to use a canonical tier-1 position (e.g., swap `coder`→`senior-dev`), or accept the subagent demotion.
3. Author `_meta.notes` explicitly noting any tier re-assignments for later review.

Decision rule for closing coverage gaps:
(a) Add canonical capabilities to existing roles if `POSITION_LIBRARY[position].canonicalCapabilities` already includes the needed capability, OR
(b) Add a minimal additional role (e.g., a `coordinator` position alongside the lead) using an appropriate provider/model.

Prefer (a) when semantically honest; use (b) otherwise.
