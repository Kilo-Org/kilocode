<task id="2" title="Implement schemas + registries + matcher + leader">
<action>

1. `src/schemas.ts` (~120 LOC) â€” Export these schemas + types matching spec Â§4 (refined):
   - `CommandScope` = `z.enum(["global", "workflow", "team-builder", "review"])`
   - `Keybind` = `z.object({ binding: z.string().min(1), leader: z.boolean().default(false) })`
   - `CommandData` = Zod-validated serializable subset: `{ id, title, description?, scope, aliases, hideKeywords, keybind?, hidden }`.
   - `Command` = TS-only interface extending `CommandData` with optional function fields: `enabled?: () => boolean` and `onSelect?: (ctx?: unknown) => void | Promise<void>`. **CRITICAL** â€” do NOT wrap `enabled` or `onSelect` in `z.function().returns(...)`. Zod 4.1.8 (catalog-pinned) removed that fluent API. Plan previously hit this in cycle-0 critique; runtime guards (`typeof cmd.onSelect === "function"`) enforce shape.
   - `CommandRegistry` interface: `register`, `unregister`, `get`, `getAllByScope`, `search`, **`subscribe(listener: () => void): () => void`** (new in refine â€” required so the SolidJS hook in Plan 03-02 Task 3 can reactively track `entries()` without polling or monkey-patching).
   - `KeybindRegistry` interface: `matchEvent({ name, ctrl, meta, shift, super?, leader }, scope): Command | undefined`.

2. `src/registry.ts` (~140 LOC):
   - `createCommandRegistry(): CommandRegistry` â€” in-memory `Map<string, Command>`. Internal `Set<() => void>` of listeners.
     - `register(cmd)`: validate serializable fields via `CommandData.parse({ ...cmd })` (spreading strips function fields before Zod sees them); throw if `id` already in map; insert; notify all listeners; return unregister fn.
     - `unregister(id)`: delete from map; notify listeners.
     - `subscribe(listener)`: add to listener set; return a function that removes it.
     - `getAllByScope(scope)`: filter by `cmd.scope === scope || cmd.scope === "global"` and `!cmd.hidden`.
     - `search(query, scope?)`: delegate to `searchCommands()` (Task 2 module 3) over visible commands.
   - `createKeybindRegistry(commands: CommandRegistry): KeybindRegistry` â€” `matchEvent(evt, scope)` iterates commands in the requested scope (plus "global"); parses each command's `keybind.binding` via internal `parseBinding` helper; returns the first matching command or undefined.

3. `src/parser.ts` (internal, ~90 LOC) â€” `parseBinding(binding: string)` MUST match the existing `packages/opencode/src/util/keybind.ts` parser byte-identical for the existing corpus:
   - Multi-combo via comma: `"ctrl+k,alt+k"` â†’ array of alternatives, any of which matches.
   - Key name normalization: `esc` â†” `escape`, `del` â†” `delete`, `space`/`spacebar`, `return` â†” `enter`.
   - Special literal `"none"` â€” returns a sentinel that matches nothing.
   - Leader prefix: `"<leader> p"` parses as `{ name: "p", leader: true, ctrl: false, meta: false, shift: false }`.
   - Modifier prefixes: `ctrl+`, `alt+`/`meta+`, `shift+`, `super+`/`cmd+`.
   - Task 3 writes a golden corpus test asserting byte-identical parse results against the existing parser's output. Keep `parseBinding` internal (no export) so Phase 5 can flip the devil-keybind impl to consume the shared opencode parser without breaking the public surface.

3. `src/matcher.ts` (~90 LOC):
   - `searchCommands(query: string, commands: Command[]): Command[]` â€” wraps `fuzzysort.go()`. Build a searchable string per command = `${title} ${aliases.join(" ")} ${hideKeywords.join(" ")}`. Sort descending by fuzzysort score; filter out items below a minimum score threshold (document the chosen value in JSDoc). Empty query returns all non-hidden commands in registration order.
   - Do NOT re-export `fuzzysort` from this module.

4. `src/leader.ts` (~80 LOC):
   - `createLeaderChain(options: { timeoutMs?: number = 2000 }): { activate(): void, press(key: string): "chained" | "reset", isActive(): boolean, cancel(): void }` â€” state machine. `activate()` starts the chain + sets a timer; if timer fires, chain resets. `press(key)` while active consumes the key (returns `"chained"`) and resets state; while inactive returns `"reset"`. `cancel()` immediately ends the chain. Flat chains only (no nesting).
   - Use `setTimeout` / `clearTimeout` â€” no platform-specific APIs. Must work in both Node/Bun and browser.
   - **INERT IN PHASE 3**: `createLeaderChain` is exported so Plan 03-02 / 03-03 can import it, but NO code in Phase 3 actually calls `activate()`. The existing Ctrl+X leader FSM in `packages/opencode/src/cli/cmd/tui/context/keybind.tsx` (lines 24â€“67) remains the live handler. Phase 5 rips out the old FSM and wires `createLeaderChain` into the new `useKeyboard`. This isolation prevents double-handler collisions with the existing TUI.

5. `src/index.ts` (~20 LOC) â€” barrel exporting schemas (`CommandData`, `Keybind`, `CommandScope`) + types (`Command`, `CommandRegistry`, `KeybindRegistry`) + factories (`createCommandRegistry`, `createKeybindRegistry`) + `searchCommands` + `createLeaderChain`. Do NOT export `parseBinding` (internal).
